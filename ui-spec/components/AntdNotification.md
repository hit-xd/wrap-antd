# Notification 通知提醒框

**中文名称：** 通知提醒框
**英文名称：** Notification
**所属分类：** 反馈
**一句话简介：** 全局展示通知提醒信息。

---

## 何时使用

当需要向用户推送需要关注的通知信息，且适合以弹出浮层方式展示完整标题和描述时使用。通知提醒框适用于后台任务完成通知、系统预警推送、客户提醒等需要传递较完整信息的场景。

该组件与 Message 全局提示的区别在于：通知提醒框支持更完整的标题+描述结构、支持自定义图标和操作按钮、可灵活控制弹出位置，适合承载信息量较大或需要用户交互的通知内容。

---

## 多状态通知

多状态通知用于展示成功、信息、警告、错误四种常见通知类型。该组件从屏幕右上角弹出浮层，自动匹配对应状态的颜色和图标，通过标题和描述共同传达完整信息。

该组件通过函数式 API 调用，由 `App.useApp()` 获取 `notification` 实例后，调用 `.success()`、`.info()`、`.warning()`、`.error()` 方法触发。

每种通知由一个卡片式浮层构成，包含左侧状态图标、标题区域、描述区域和右上角关闭按钮。

---

### 示例状态

- **成功通知**
  用于操作已成功完成的通知，例如「客户提醒已发送」。左侧图标为绿色圆形勾选图标。

- **信息通知**
  用于中性信息告知，例如「经营建议更新，系统已生成新的资产配置建议」。左侧图标为蓝色圆形信息图标。

- **警告通知**
  用于需要用户关注的通知，例如「仍有 2 位客户待补录 KYC 资料」。左侧图标为黄色三角形感叹号图标。

- **错误通知**
  用于操作失败或异常的通知，例如「数据同步异常，请检查网络后重试」。左侧图标为红色圆形叉号图标。

---

## 多状态通知样式

### 整体样式

通知提醒框宽度为 `384px`，高度自适应。水平内边距为 `24px`，垂直内边距为 `16px`，容器圆角为 `4px`。

多个通知同时存在时，相邻通知之间的垂直间距为 `16px`。

通知提醒框从屏幕右上角弹出，点击右上角关闭按钮或达到自动关闭时间后消失。

---

### 标题样式

标题文字使用字号为 `16px`，字重为 `Bold`，文字颜色为 `#000000`，透明度为 `85%`。

标题与描述文字之间的垂直间距为 `4px`。

---

### 描述样式

描述文字使用字号为 `14px`，字重为 `Regular`，文字颜色为 `#000000`，透明度为 `65%`，行高为 `1.6`。

---

### 状态色规范

| 类型 | 图标色 | 触发按钮色 |
|------|--------|-----------|
| success | `#00C078` | 绿色边框按钮 |
| info | `#1874FF` | 蓝色边框按钮 |
| warning | `#FFA000` | 橙色边框按钮 |
| error | `#FF4D4F` | danger 按钮 |

## 指定弹出位置

通过 `placement` 属性控制通知出现的屏幕位置，适应不同业务场景的视觉布局需求。

支持六种弹出位置：右上角（默认）、右下角、左上角、左下角、顶部居中、底部居中。

默认弹出位置为右上角（`topRight`），遵循用户浏览页面的视觉动线，将通知放置在视觉起点附近。

底部弹出位置（`bottomRight`、`bottomLeft`、`bottom`）适合页面顶部已有固定导航栏或公告的场景，避免通知与页面头部信息重叠。

---

## 自定义图标

当默认状态图标不能满足业务语义时，可使用 `icon` 属性自定义通知图标。

自定义图标应保持尺寸与默认图标一致，颜色建议使用品牌色或状态色，确保在通知卡片中清晰可辨。

自定义图标通过 `notification.open()` 方法传入 `icon` 属性实现，适合需要强调特定业务含义的通知场景。

---

## 自定义按钮

当通知需要引导用户执行具体操作时使用，例如「立即查看」「去处理」「查看详情」等。

通过 `btn` 属性在通知卡片底部添加操作按钮，适合需要用户响应的通知场景。

操作按钮使用 `WplusButton size="small" type="primary"` 样式，避免在通知区域中引入过高视觉权重的交互。

建议配合 `duration: 0` 使用，使通知持续显示，等待用户操作后再手动关闭。

---

## 纯文字提醒框

当不需要左侧状态图标，仅展示文字信息时使用。

通过 `notification.open()` 方法直接传入 `message` 和 `description` 参数，不指定 `icon` 属性，通知卡片将不显示左侧图标区域，仅展示标题和描述文本。

纯文字提醒框适合系统推送、运营公告等不需要强调状态属性的通知内容。

---

## 交互规则

### 调用方式

通知提醒框通过 `App.useApp()` 获取 `notification` 实例后调用：

- `notification.success({ message, description })` — 成功通知
- `notification.info({ message, description })` — 信息通知
- `notification.warning({ message, description })` — 警告通知
- `notification.error({ message, description })` — 错误通知
- `notification.open({ message, description, icon, btn, placement, duration })` — 通用调用

### 自动关闭

通知默认 `4.5` 秒后自动关闭。可通过 `duration` 参数自定义显示时长，设置为 `0` 时不自动关闭，需手动点击关闭按钮或调用 `notification.destroy()` 关闭。

## 使用场景

**适用场景：**
- 后台任务完成通知，例如数据同步完成、报表生成完毕
- 系统预警推送，例如风险等级变更、资料补录提醒
- 客户提醒与待办事项推送
- 需要展示完整标题和描述的通知信息

**不适用场景：**
- 页面内的轻量提示（请使用 Message 全局提示）
- 页面内静态提示信息（请使用 Alert 警告提示）
- 需要用户确认的操作（请使用 Modal 对话框）

---

## 注意事项

1. 通知提醒框标题应简明扼要，控制在 15 字以内；描述用于补充说明背景信息。
2. 默认弹出位置为右上角，请根据页面布局合理选择 `placement`。
3. 自定义按钮建议使用 `duration: 0` 使通知持续显示，待用户操作后主动关闭。
4. 不建议在同一时间弹出超过 3 条通知，避免遮挡页面主要操作区域。
5. 纯文字提醒框适合无需强调状态属性的通知，如需展示状态请使用多状态通知。
6. 通知提醒框与 Message 全局提示的区别在于：Notification 支持更完整的标题+描述结构，Message 更轻量。
