# Layout 布局

**中文名称：** 布局  
**英文名称：** Layout  
**所属分类：** 布局  
**一句话简介：** 页面整体骨架，包含顶部导航、侧边栏、内容区和页脚等容器；W+ 私行体系主要提供三种核心布局骨架：**顶侧布局（经典后台）**、**首页版心布局**（门户风格）和**三层顶部导航布局**（多子系统导航结构）。

---

## 何时使用

当需要快速搭建页面整体骨架时使用。布局组件提供顶部导航、侧边栏、内容区和页脚等容器，用于构建标准业务页面的整体框架。

W+ 私行推荐业务页面优先使用 `BusinessPageLayout` 业务布局组件，它内置了顶部导航菜单、侧边导航菜单和内容区的标准骨架。

> 当产品存在**多个子系统/工作台**需要全局通栏承载切换，且子系统内模块偏平（核心 ≤ 2 级）、以横向切换为主时，可改用 **三层顶部导航布局**（全局通栏 → 二级横排导航 → 三级 Tabs/页头），见下方对应章节。

---

## 顶部-侧边布局（经典后台）

顶部-侧边布局是最常用的后台管理系统标准骨架，适用于大多数业务页面，包括列表页、审批页、详情页等。

> **适用角色：** 私行 W+ 体系中的**客户经理**角色。该角色日常使用频次高、操作路径深，需要侧边栏承载纵深树状菜单以便快速定位模块。

该布局由三部分组成：顶部导航栏、左侧侧边栏和内容区域。顶部导航承载全局功能切换，侧边栏承载模块内导航，内容区自适应剩余空间。

## 首页版心布局（通栏 Header + 居中内容画布）

### 何时使用

- 系统入口页 / 工作台首页，需要**稳住视觉重心**，不让内容在大屏上被拉散
- 首页以**概览卡片、快捷入口、公告横幅、关键指标 KPI**为主，而非密集表格/表单
- Header 为全宽通栏（Logo、主导航、用户区），**内容区有明确的"画布边界"**，以便对内做栅格/卡片对齐

> **适用角色：** 私行 W+ 体系中的**所有角色**（客户经理、中心负责人、客群经理等）。首页版心布局作为工作台首页的通用骨架，不分角色均可使用。

### 结构拆分（两层：全宽 Header + 版心画布）

#### ① Header — 全宽通栏

Header 为全宽通栏设计，内部元素（Brand + 导航 + 用户区）直接采用全宽布局。

| 属性 | 数值 |
|------|------|
| 高度 | `64px` |
| 背景色 | `#090a15` |
| 文字颜色 | `#c7c9d3` |
| 内边距 | `0 24px` |
| 布局 | flex，`align-items: center` |

**Brand 品牌区：**

| 元素 | 样式 |
|------|------|
| 容器宽度 | `172px`，右边距 `26px` |
| "W" 字符 | 颜色 `#e9b35d`（金色），字号 `30px`，字重 `900`，字体 `Georgia, serif` |
| "+" 字符 | 颜色 `#e9b35d`（金色），字号 `21px` |
| "私行" 文字 | 颜色 `#fff`（白色），字号 `23px`，字重 `700`，字间距 `-1px` |

**导航项：**

| 属性 | 数值 |
|------|------|
| 字号 | `15px` |
| 最小宽度 | `78px` |
| 内边距 | `0 13px` |
| 默认状态 | 文字 `#c7c9d3`，字重 `400` |
| 选中状态 | 文字 `#fff`，字重 `600`，底部 `3px solid #f4cd74` 指示器（圆角 `2px 2px 0 0`，左右缩进 `13px`） |
| "更多"项 | 文字 `#a9acb8`，后缀 `⌄` |
| 数量 | 通常 7 个（含"更多"） |

**用户区：**

| 属性 | 数值 |
|------|------|
| 容器宽度 | `214px`，flex `justify-content: flex-end`，间距 `10px` |

| 用户名 | 字号 `14px`，颜色 `#fff` |
| 副标题 | 字号 `12px`，颜色 `#bfc3d0` |

#### ② 版心内容画布（Content Canvas / Page Margin Box）

| 属性 | 数值 |
|------|------|
| **左右留白（页边距）** | 固定侧留白 **`261px`**（落实到代码通常表达为 `padding: 0 261px` 或 `width: calc(100% - 522px)` + `margin: auto`） |
| 版心宽度计算 | `版心宽 = 100vw - 261px × 2`（在 `1440px` 设计稿上版心 ≈ `918px`；在 `1920px` 宽屏上版心 ≈ `1398px`） |
| 顶部偏移 | `40px`（距 Header 底边缘） |
| 底部边距 | `48px` |
| 背景色 | `#F5F6F7` |

### 响应式规则

| 断点 | 左右留白策略 | 说明 |
|------|------------|------|
| `≥ 1600px` | `261px`（或按比例微调到 `280~320px`） | 大屏保持"画廊感" |
| `1440px ~ 1599px` | `261px`（设计稿锚点区间） | 保持设计还原 |
| `1280px ~ 1439px` | 留白收敛为 `80px ~ 120px` | 防止版心被压到不可读 |
| `< 1280px` | 切为 `padding: 0 24px`（同业务页边距体系） | 此时应评估是否还用"首页版心"——可能降级为普通 `BusinessPageLayout` 的首页变体 |

## 三层顶部导航布局（全局通栏 + 二级横排导航 + 三级 Tabs）

### 何时使用

- 多子系统/工作台控制台
- 模块扁平且需要高频横向切换的业务系统
- 需要全局通栏承载跨系统切换的场景

> **适用角色：** 私行 W+ 体系中**客户经理以外的角色**，例如**中心负责人**（管理多个团队/模块，需要全局视角频繁切换子系统）、**客群经理**（面向多客群维度，需要在不同业务模块间横向跳转）等。该类角色的工作特征为：模块扁平、跨度大、切换频次高，侧边栏纵深结构无法满足横向切换的效率需求。

### 三层结构设计规范

#### Layer 1 - 全局通栏

| 属性 | 数值 |
|------|------|
| 高度 | `64px` |
| 背景色 | `#090a15` |
| 文字颜色 | `#c7c9d3` |
| 内边距 | `0 24px` |
| 定位 | flex，`align-items: center` |

L1 与首页版心的 Header 共用相同的 Brand（"W+ 私行"）和导航项（15px、`#c7c9d3`/`#fff`、`#f4cd74` 指示器）样式规范，详见首页版心布局章节。

#### Layer 2 - 二级模块导航（参考 WplusPrimaryTabs 一级标签页纯文字）

| 容器背景 | `#fff` |
| 容器底部边框 | `1px solid rgba(0, 0, 0, 0.12)` |
| 对齐方式 | `align-items: flex-end` |
| 标签高度 | `48px`（撑满容器） |
| 标签最小宽度 | `86px` |
| 标签间距 | `24px`（首个为 `0`） |
| 标签字号 | `14px` |

### 二级导航溢出收展方案

当二级导航项超过 7-8 个时，需要考虑溢出收展方案：

| 方案 | 适合超出规模 | 推荐度 | 说明 |
|------|------------|--------|------|
| 静态「更多 ▾」下拉 + 业务分组 | 8～12 个 | ⭐⭐⭐⭐ | 人工优先级定级，核心常驻露出 |
| 动态溢出折叠（ResizeObserver） | 8～15 个 | ⭐⭐⭐ | 运行时按可用宽度自动收展 |
| 「所有模块」Drawer / Mega Panel | ≥12 个 | ⭐⭐⭐⭐⭐ | 持续增长时的最优解 |
| 横滚 + 渐隐箭头 | 更像 Tab 集合 | ⭐⭐ | 谨慎使用，仅适合视图集合 |

## 使用场景对比

| 布局类型 | 适用场景 | 适用角色 | 不适用场景 |
|----------|----------|----------|------------|
| 顶部-侧边布局 | 纵深树状菜单业务系统 | 客户经理 | 模块扁平需要高频横向切换 |
| 首页版心布局 | 工作台首页、门户页面 | 所有角色 | 业务 CRUD 页面、带侧边栏页面 |
| 三层顶栏布局 | 多子系统控制台、扁平模块系统 | 中心负责人、客群经理等（客户经理除外） | 单系统纵深菜单、简单页面 |

## 注意事项

1. **布局选择原则**
   - 同一子系统内保持布局一致性
   - 根据页面类型和用户场景选择合适的布局
   - 不要混用不同布局哲学

2. **响应式处理**
   - 所有布局都必须考虑响应式适配
   - 首页版心的 261px 留白需要响应式降级
   - 三层顶栏在小屏下应考虑降级方案

3. **导航溢出处理**
   - 二级导航项超过 7-8 个时需考虑收展方案
   - 模块持续增长时优先使用「所有模块」面板
   - 避免硬扛横向空间导致体验问题

4. **可访问性**
   - 深色通栏必须确保对比度合规（WCAG AA 4.5:1）
   - 所有交互元素必须支持键盘导航
   - 导航状态必须明确标识

5. **性能优化**
   - 动态溢出折叠需要防抖处理
   - 大屏留白应考虑按需加载
   - 复杂布局应考虑懒加载策略
