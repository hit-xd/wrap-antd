# 5.1 大尺寸水平方向步骤条样式（横向 32px）

## 1. 组件定义

大尺寸水平方向步骤条是页面级流程进度组件，基于 Ant Design `Steps` 横向模式二次封装。

## 2. 使用场景

### 2.1 大尺寸水平方向步骤条（横向 32px）

用于页面级主流程展示，例如办理流程、审批流程、开户流程、申请流程等。

适合场景：

- 页面顶部的主流程进度；
- 用户需要明确知道当前处于第几步；
- 步骤数量较少，通常 3-5 步；
- 每个步骤需要同时展示标题和描述信息。

不适合场景：

- 表格内部或弹窗内部的紧凑流程；
- 只用于切换内容视图的场景；

## 3. 结构组成

### 3.1 横向步骤条结构

横向步骤条由以下部分组成：

- 步骤圆点 / 图标区；
- 步骤标题；
- 步骤描述；
- 横向连接线；
- 已完成状态、进行中状态、未开始状态。

标题与描述为左对齐，文字区域位于圆点右侧。连接线从步骤文字区域右侧延展至下一步骤。

- 32px 步骤圆点；
- 步骤标题；
- 步骤描述；
- 1px 横向连接线；
- 已完成、进行中、未开始三种状态。

## 4. 状态规则

### 4.1 常规步骤条状态

常规步骤条仅包含以下三种状态：

1. 已完成状态
2. 进行中状态
3. 未开始状态

不要额外拆出“最后一步状态”。最后一个步骤如果样式不同，只作为布局规则处理，例如是否显示后方连接线。

### 4.2 已完成状态

已完成状态表示该步骤已完成。

视觉规则：

- 圆点为透明背景；
- 圆点使用品牌色描边；
- 圆点内部展示完成图标；
- 该步骤后方连接线使用已完成连接线颜色。

### 4.3 进行中状态

进行中状态表示用户当前所在步骤。

视觉规则：

- 圆点为品牌色实心背景；
- 圆点内部展示白色数字；
- 标题文字为当前强调状态；
- 该步骤后方连接线使用未完成连接线颜色。

### 4.4 未开始状态

未开始状态表示尚未进入的后续步骤。

视觉规则：

- 圆点为透明背景；
- 圆点使用弱化描边；
- 圆点内部数字使用弱化文字色；
- 标题和描述使用弱化文字色；
- 后方连接线使用未完成连接线颜色。

# 5.5 导航步骤条样式 - 无 icon

## 1. 组件定义

导航步骤条无 icon 样式用于流程模块之间的横向导航，组件名称建议为 `PrivateArrowSteps`。

该组件不是 Ant Design 默认 `Steps` 的圆点步骤条，也不是普通按钮组、Tabs 或 Segmented。不要使用 Ant Design Steps 的 DOM 结构，不允许生成圆点、圆形数字、普通横向连接线。

这个组件本质是一个「横向模块化流程导航条」，它由多个固定尺寸的模块步骤基础单元横向叠放拼接而成。视觉上看起来像连续的流程条，而不是多个彼此分离的箭头按钮。

组件由多个步骤模块组成，每个步骤包含一个主矩形区域。步骤之间的箭头感不是通过 margin / gap 形成的，也不是把每个步骤整体做成 chevron 六边形，而是通过以下结构叠放实现：

1. 每个步骤都有一个 `134px × 48px` 的主矩形区域；
2. 非最后一个步骤右侧有一个 `12px` 宽的右三角箭头片；
3. 除第一个步骤外，每个步骤左侧有一个 `12px` 宽的 V 形覆盖片，用于压在前一个步骤的右箭头区域上；
4. 后一个步骤要覆盖在前一个步骤上方，形成连续的模块拼接效果。

---

## 2. 使用场景

导航步骤条无 icon 样式用于强调流程模块推进关系的横向导航型步骤条。

适合场景：

- 业务办理模块切换；
- 流程分段导航；
- 办理流程、审批流程、配置流程或业务节点流转；
- 需要箭头形态强化先后关系，但不需要展示成功 / 错误状态的场景；
- 仅通过背景色和文字颜色表达当前所在步骤的流程导航。

不适合场景：

- Ant Design 默认圆点步骤条；
- Tabs 标签页；
- Segmented 分段控制器；
- ButtonGroup 按钮组；
- 普通卡片式步骤按钮。

整体高度为 `48px`，宽度根据步骤数量自适应。

2. `2.已完成步骤`


请按照以下方式绘制每个步骤模块。


推荐 SVG path：

```ts
L x+134 48
H x-12
```

其中：


最后一个步骤左侧有 V 形覆盖片，右侧是平直边，不再显示右箭头。

```ts
H x+134
```

其中：


必须按步骤顺序从左到右渲染，并且后一个步骤要覆盖在前一个步骤上方。

不要把每个步骤做成彼此独立的卡片。

不要让步骤之间出现明显空白。  
不要把每个步骤都做成完整的 chevron 六边形。

正确效果应该是：后一项的左侧 V 形覆盖片叠在前一项右箭头区域上，形成连续拼接。

### 5.2 推荐实现方式

推荐使用 SVG 实现整体步骤条。

外层 SVG：

```tsx
<svg
  width={computedWidth}
  height={48}
  viewBox={`0 0 ${computedWidth} 48`}
>
  {steps.map((step, index) => (
    <g
      key={step.key}
      className="private-arrow-step"
      role="button"
      tabIndex={step.disabled ? -1 : 0}
      aria-disabled={step.disabled}
      onClick={() => handleStepClick(step)}
      onKeyDown={(event) => handleStepKeyDown(event, step)}
    >
      <path
        d={getStepPath(index, steps.length)}
        fill={getStepFill(step.status)}
      />
      <path
        d={getStepPath(index, steps.length)}
        fill="transparent"
        className="private-arrow-step-hit-area"
      />
      <foreignObject
        x={index * 150}
        y={0}
        width={134}
        height={48}
        pointerEvents="none"
      >
        <div className="private-arrow-step-label">
          {step.label}
        </div>
      </foreignObject>
    </g>
  ))}
</svg>
```

路径生成规则：

```ts
const BODY_WIDTH = 134;
const STEP_INTERVAL = 150;
const ARROW_WIDTH = 12;
const HEIGHT = 48;
const MID_Y = 24;

function getStepPath(index: number, total: number) {
  const x = index * STEP_INTERVAL;

  if (total === 1) {
    return `M ${x} 0 H ${x + BODY_WIDTH} V ${HEIGHT} H ${x} Z`;
  }

  if (index === 0) {
    return `
      M ${x} 0
      H ${x + BODY_WIDTH}
      L ${x + BODY_WIDTH + ARROW_WIDTH} ${MID_Y}
      L ${x + BODY_WIDTH} ${HEIGHT}
      H ${x}
      Z
    `;
  }

  if (index === total - 1) {
    return `
      M ${x - ARROW_WIDTH} 0
      H ${x + BODY_WIDTH}
      V ${HEIGHT}
      H ${x - ARROW_WIDTH}
      L ${x} ${MID_Y}
      Z
    `;
  }

  return `
    M ${x - ARROW_WIDTH} 0
    H ${x + BODY_WIDTH}
    L ${x + BODY_WIDTH + ARROW_WIDTH} ${MID_Y}
    L ${x + BODY_WIDTH} ${HEIGHT}
    H ${x - ARROW_WIDTH}
    L ${x} ${MID_Y}
    Z
  `;
}
```

宽度计算规则：

```ts
const computedWidth = (steps.length - 1) * 150 + 134;
```

4 步时：

```ts
computedWidth = 3 * 150 + 134 = 584;
```

### 5.5 交互规则

该组件必须支持点击交互，不应只是静态 SVG。

每个步骤都必须是一个可点击项，点击任意步骤的完整模块区域时，都应触发步骤切换事件。

点击区域要求：

- 点击区域必须覆盖当前步骤完整视觉区域；
- 点击区域包括主矩形区域、左侧 V 形覆盖片、右侧三角箭头片；
- 不能只让文字可点击；
- 不能只让 `path` 的有色区域可点击；
- 不能因为 `foreignObject`、文字层或 SVG 层级导致点击失效；
- 鼠标悬停在可点击步骤上时，光标应显示为 `pointer`。

组件需要支持受控和非受控两种模式：

```ts
activeKey?: string;
defaultActiveKey?: string;
onChange?: (key: string, item: ArrowStepItem) => void;
```

点击逻辑：

1. 判断该步骤是否 disabled；
2. 如果 disabled，则不触发任何事件；
3. 如果不是 disabled，则触发 `onChange(step.key, step)`；
4. 如果组件处于非受控模式，则内部更新 activeKey；
5. 当前 activeKey 对应的步骤展示进行中状态。

`foreignObject` 或文字层需要设置 `pointer-events: none`，避免挡住点击。

键盘交互要求：

- 每个非禁用步骤需要支持键盘聚焦；
- 按 Enter 或 Space 时，应触发与点击相同的切换逻辑；
- 禁用步骤不参与键盘聚焦；
- 聚焦态样式使用组件库统一 focus 样式，若无统一规范，先保留默认浏览器 focus 或使用金色描边。

9. 页面中没有圆点、圆形数字、普通连接线。
10. 文字居中在主矩形区域内，而不是居中在整个箭头外轮廓内。
11. 点击步骤文字可以切换。
12. 点击步骤空白区域也可以切换。
13. 点击左侧 V 形覆盖片可以切换。
14. 点击右侧三角箭头片可以切换。
15. 文字层不会阻挡 SVG 点击事件。


# 5.6 导航步骤条样式 - 有 icon

## 1. 组件定义

导航步骤条有 icon 样式用于在横向流程导航中展示步骤状态，例如成功、错误、校验通过、校验失败等，组件名称建议为 `PrivateArrowSteps`，通过状态 icon 配置展示有 icon 版本。

该组件不是 Ant Design 默认 `Steps` 的圆点步骤条，也不是普通按钮组、Tabs 或 Segmented。不要使用 Ant Design Steps 的 DOM 结构，不允许生成圆点、圆形数字、普通横向连接线。

这个组件本质是一个「横向模块化流程导航条」，它由多个固定尺寸的模块步骤基础单元横向叠放拼接而成。视觉上看起来像连续的流程条，而不是多个彼此分离的箭头按钮。

组件由多个步骤模块组成，每个步骤包含一个主矩形区域。步骤之间的箭头感不是通过 margin / gap 形成的，也不是把每个步骤整体做成 chevron 六边形，而是通过以下结构叠放实现：

1. 每个步骤都有一个 `134px × 48px` 的主矩形区域；
2. 非最后一个步骤右侧有一个 `12px` 宽的右三角箭头片；
3. 除第一个步骤外，每个步骤左侧有一个 `12px` 宽的 V 形覆盖片，用于压在前一个步骤的右箭头区域上；
4. 后一个步骤要覆盖在前一个步骤上方，形成连续的模块拼接效果；
5. 有 icon 版本中，状态 icon 位于文字右侧，文字与 icon 作为一个整体居中显示。

---

## 2. 使用场景

导航步骤条有 icon 样式用于在横向流程导航中展示步骤状态，例如成功、错误、校验通过、校验失败等。

适合场景：

- 分段表单校验流程；
- 每一步可能出现成功或错误状态；
- 需要在流程导航中直观看到状态反馈；
- 业务办理模块切换；
- 流程分段导航；
- 需要箭头形态强化先后关系，并且需要展示状态 icon 的流程导航。

不适合场景：

- Ant Design 默认圆点步骤条；
- Tabs 标签页；
- Segmented 分段控制器；
- ButtonGroup 按钮组；
- 普通卡片式步骤按钮；
- icon 位于文字前方的步骤条。

---

## 3. 结构组成

### 3.1 整体结构

整体高度为 `48px`，宽度根据步骤数量自适应。

当前示例为 4 个步骤：

1. `1.已完成步骤`，文字右侧展示成功 icon；
2. `2.已完成步骤`，文字右侧展示成功 icon；
3. `3.进行中步骤`，文字右侧展示错误 icon；
4. `3.未开始步骤`，不展示 icon。

整体示例宽度为 `584px`。

尺寸规则：

- 步骤主矩形宽度：`134px`
- 步骤高度：`48px`
- 相邻步骤主矩形起始位置间隔：`150px`
- 右侧三角箭头片宽度：`12px`
- 左侧 V 形覆盖片宽度：`12px`
- 4 步时整体宽度：`584px`

步骤主矩形起始位置规则：

```ts
const STEP_BODY_WIDTH = 134;
const STEP_INTERVAL = 150;
const ARROW_WIDTH = 12;
const STEP_HEIGHT = 48;

const bodyX = index * STEP_INTERVAL;
```

4 个步骤的主矩形起始位置分别为：

```txt
第 1 步：x = 0
第 2 步：x = 150
第 3 步：x = 300
第 4 步：x = 450
```

注意：这里的 `150px` 不是步骤之间的空白间距，而是每个步骤主矩形的起始位置间隔。不要使用 `gap: 16px` 或 `margin-left: 16px` 把步骤分开。

### 3.2 模块几何结构

请按照以下方式绘制每个步骤模块。

#### 第一个步骤

第一个步骤左侧是平直边，右侧有三角箭头片。

推荐 SVG path：

```ts

H x+134
L x+146 24
L x+134 48
H x
Z
```

其中：

```ts

```

#### 中间步骤

中间步骤左侧有 V 形覆盖片，右侧有三角箭头片。

推荐 SVG path：

```ts
M x-12 0
H x+134
L x+146 24
L x+134 48
H x-12
L x 24
Z
```

其中：

```ts
x = index * 150
```

#### 最后一个步骤

最后一个步骤左侧有 V 形覆盖片，右侧是平直边，不再显示右箭头。

推荐 SVG path：

```ts
M x-12 0
H x+134
V 48
H x-12
L x 24
Z
```

其中：

```ts
x = index * 150
```

### 3.3 渲染顺序

必须按步骤顺序从左到右渲染，并且后一个步骤要覆盖在前一个步骤上方。

不要把每个步骤做成彼此独立的卡片。  
不要使用 `display: flex; gap: 16px` 来排列步骤。  
不要让步骤之间出现明显空白。  
不要把每个步骤都做成完整的 chevron 六边形。

正确效果应该是：后一项的左侧 V 形覆盖片叠在前一项右箭头区域上，形成连续拼接。

### 3.4 文字与 icon 结构

有 icon 版本中，文字与 icon 水平排列，作为一个内容组展示在步骤主矩形区域中。

结构包括：

- 步骤文字；
- 状态 icon；
- 文字与 icon 的间距。

要求：

- icon 位于文字右侧；
- 文字与 icon 作为一个整体水平居中；
- icon 不得放在文字前方；
- icon 不得贴在步骤块最右侧；
- 文字区域仍然跟随主矩形起始位置，不要把文字与 icon 放到整个箭头形状的几何中心。

### 5.2 推荐实现方式

推荐使用 SVG 实现整体步骤条。

外层 SVG：

```tsx
<svg
  width={computedWidth}
  height={48}
  viewBox={`0 0 ${computedWidth} 48`}
>
  {steps.map((step, index) => (
    <g
      key={step.key}
      className="private-arrow-step"
      role="button"
      tabIndex={step.disabled ? -1 : 0}
      aria-disabled={step.disabled}
      onClick={() => handleStepClick(step)}
      onKeyDown={(event) => handleStepKeyDown(event, step)}
    >
      <path
        d={getStepPath(index, steps.length)}
        fill={getStepFill(step.status)}
      />
      <path
        d={getStepPath(index, steps.length)}
        fill="transparent"
        className="private-arrow-step-hit-area"
      />
      <foreignObject
        x={index * 150}
        y={0}
        width={134}
        height={48}
        pointerEvents="none"
      >
        <div className="private-arrow-step-label private-arrow-step-label-with-icon">
          <span>{step.label}</span>
          {step.icon ? <span className="private-arrow-step-icon">{step.icon}</span> : null}
        </div>
      </foreignObject>
    </g>
  ))}
</svg>
```

路径生成规则：

```ts
const BODY_WIDTH = 134;
const STEP_INTERVAL = 150;
const ARROW_WIDTH = 12;
const HEIGHT = 48;
const MID_Y = 24;

function getStepPath(index: number, total: number) {
  const x = index * STEP_INTERVAL;

  if (total === 1) {
    return `M ${x} 0 H ${x + BODY_WIDTH} V ${HEIGHT} H ${x} Z`;
  }

  if (index === 0) {
    return `
      M ${x} 0
      H ${x + BODY_WIDTH}
      L ${x + BODY_WIDTH + ARROW_WIDTH} ${MID_Y}
      L ${x + BODY_WIDTH} ${HEIGHT}
      H ${x}
      Z
    `;
  }

  if (index === total - 1) {
    return `
      M ${x - ARROW_WIDTH} 0
      H ${x + BODY_WIDTH}
      V ${HEIGHT}
      H ${x - ARROW_WIDTH}
      L ${x} ${MID_Y}
      Z
    `;
  }

  return `
    M ${x - ARROW_WIDTH} 0
    H ${x + BODY_WIDTH}
    L ${x + BODY_WIDTH + ARROW_WIDTH} ${MID_Y}
    L ${x + BODY_WIDTH} ${HEIGHT}
    H ${x - ARROW_WIDTH}
    L ${x} ${MID_Y}
    Z
  `;
}
```

宽度计算规则：

```ts
const computedWidth = (steps.length - 1) * 150 + 134;
```

4 步时：

```ts
computedWidth = 3 * 150 + 134 = 584;
```

### 5.5 交互规则

该组件必须支持点击交互，不应只是静态 SVG。

每个步骤都必须是一个可点击项，点击任意步骤的完整模块区域时，都应触发步骤切换事件。

点击区域要求：

- 点击区域必须覆盖当前步骤完整视觉区域；
- 点击区域包括主矩形区域、左侧 V 形覆盖片、右侧三角箭头片；
- 不能只让文字或 icon 可点击；
- 不能只让 `path` 的有色区域可点击；
- 不能因为 `foreignObject`、文字层、icon 层或 SVG 层级导致点击失效；
- 鼠标悬停在可点击步骤上时，光标应显示为 `pointer`。

组件需要支持受控和非受控两种模式：

```ts
activeKey?: string;
defaultActiveKey?: string;
onChange?: (key: string, item: ArrowStepItem) => void;
```

点击逻辑：

1. 判断该步骤是否 disabled；
2. 如果 disabled，则不触发任何事件；
3. 如果不是 disabled，则触发 `onChange(step.key, step)`；
4. 如果组件处于非受控模式，则内部更新 activeKey；
5. 当前 activeKey 对应的步骤展示进行中状态。

`foreignObject`、文字层和 icon 层需要设置 `pointer-events: none`，避免挡住点击。

键盘交互要求：

- 每个非禁用步骤需要支持键盘聚焦；
- 按 Enter 或 Space 时，应触发与点击相同的切换逻辑；
- 禁用步骤不参与键盘聚焦；
- 聚焦态样式使用组件库统一 focus 样式，若无统一规范，先保留默认浏览器 focus 或使用金色描边。
