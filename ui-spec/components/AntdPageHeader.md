# PageHeader

**中文名称：** 页头
**英文名称：** PageHeader
**所属分类：** 导航组件
**一句话简介：** 用于详情页面顶部，统一承载「面包屑 + 对象标题/副标题 + 页面级操作按钮 + 对象关键描述字段」，帮助用户在进入页面即刻识别对象身份并触达主要操作。

---

> 适用范围：本规范用于基于 Ant Design 搭建的 npm 组件库中，调整「详情页页头 / PageHeader」组件的视觉样式与组合规则。  
> 组件库可基于 Ant Design 的 `Breadcrumb`、`Button`、文本排版能力和描述列表能力进行二次封装，但业务页面应优先使用封装后的详情页页头组件，不建议在业务页面中临时拼接面包屑、标题、按钮和描述信息。

---

## 1. 组件定义

详情页页头用于详情页面顶部，承载当前对象的基础信息、页面标题、辅助说明、页面级操作按钮以及关键描述字段。

该组件主要帮助用户快速识别当前详情页对象，并提供与当前对象相关的主要操作入口。

组件由三部分组成：

1. 面包屑区域
2. 标题及操作区域
3. 描述列表区域

---

## 2. 使用场景

适用于以下场景：

- 家族信托详情页页头
- 需要展示当前页面路径、当前对象标题、对象摘要信息和页面级操作按钮的页面。
- 页面顶部需要统一呈现「面包屑 + 标题 + 操作 + 描述字段」的场景。

不建议用于以下场景：

- 普通查询列表页。
- 数据看板首页。
- 只需要简单标题、不需要对象描述信息的页面。
- 弹窗内部的轻量标题区域。

---

## 3. 结构组成

### 3.1 外层容器

详情页页头整体为一个白色内容容器。

结构顺序为：

1. 面包屑
2. 标题及操作按钮
3. 描述列表

页面元素距离容器左右边距为 `20px`。

当前示例中，组件容器宽度为 `1000px`，但实际使用时应支持宽度自适应。

### 3.2 面包屑区域

面包屑位于页头内容顶部，用于展示当前页面路径。

示例内容：

```text
首页 / 页面二 / 当前页
```

面包屑区域位于标题及操作区域上方。

### 3.3 标题及操作区域

标题及操作区域由左侧标题信息和右侧操作按钮组成。

左侧标题信息包括：

- 主标题
- 副标题或标题描述

右侧操作区域包括：

- 次按钮
- 主按钮

按钮最多展示 `4` 个。

主按钮固定在最右侧，其余按钮位于主按钮左侧，并按照操作重要程度从右向左或从近到远排列。

### 3.4 描述列表区域

描述列表用于展示当前详情对象的关键字段。

示例字段包括：

```text
详情页类型：分页详情
设计等级：较难
更新人：张三/23456723
更新时间：2025-06-09 18:32
```

描述列表由多条描述项组成，每条描述项包含：

- 描述项标题 / Label
- 描述项内容 / Value

---

## 4. 状态规则

### 4.1 基础状态

详情页页头默认展示完整结构：面包屑、标题及操作区域、描述列表。

如果某一业务页面不需要副标题、按钮或描述列表，可以按业务配置隐藏，但隐藏后不应破坏其他区域的间距关系。

### 4.2 按钮状态

操作按钮分为次按钮和主按钮。

次按钮使用 Ant Design 默认次按钮样式，并按本组件规范调整尺寸、间距和圆角。

主按钮使用品牌主色填充样式，放置在操作区最右侧。

按钮数量规则：

- 最多展示 `4` 个按钮。
- 主按钮最多 `1` 个，位于最右侧。
- 次按钮位于主按钮左侧。
- 如果按钮过多，需要由业务侧收敛，不建议在页头区域横向堆叠过多操作。

### 4.3 描述项状态

描述项默认展示为「标题 + 内容」的横向文本结构。

描述项内容为空时的占位规则：展示 --

描述项内容过长时的换行或省略规则：最后一个字符展示省略号...

---

## 5. 尺寸 / 间距 / 颜色 / 交互规则

## 5.1 外层容器样式

整体容器宽度：自适应，当前示例宽度为 `1000px`  
整体容器高度：根据内容自适应，当前示例高度为 `174px`  
整体容器背景色：`#FFFFFF`  
整体容器边框：无描边  
整体容器圆角：`0px`  
整体容器阴影：无明显阴影  

容器左右内边距：`20px`  
容器顶部内边距：`20px`  
容器底部内边距： `20px`  

内容区域宽度：容器宽度减去左右内边距，当前示例为 `960px`

---

## 5.2 面包屑区域样式

直接调用面包屑组件

面包屑区域高度：`22px`  
面包屑与标题及操作区域之间的垂直间距：`12px`  

---

## 5.3 标题及操作区域样式

标题及操作区域高度：`32px`  
标题及操作区域宽度：自适应，当前示例内容区域宽度为 `960px`  
标题及操作区域布局：左侧标题信息，右侧操作按钮  
标题区域与按钮区域垂直居中对齐  

标题及操作区域与描述列表之间的垂直间距：`16px`

### 主标题样式

主标题文字颜色：`#000000 85%`  
主标题字号：`20px`  
主标题字重：`SemiBold`  
主标题行高：`28px`  
主标题字间距：`0px`  

### 副标题样式

副标题文字颜色：`#000000 65%`  
副标题字号：`14px`  
副标题字重：`Regular`  
副标题行高：`22px`  
副标题字间距：`0px`  

标题与副标题之间的水平间距：`16px`  
标题与副标题文字左对齐方式：水平居中对齐

### 操作按钮区域样式
直接调用高度为32px的主按钮与次按钮

按钮最多展示数量：`4` 个  
主按钮位置：最右侧  
次按钮位置：主按钮左侧，按操作重要程度依次排列  
按钮之间的水平间距：`12px`  

基于 Ant Design 调整时：

- 次按钮可基于 `Button` 默认类型封装。
- 主按钮可基于 `Button type="primary"` 封装，但需要覆盖主色为 `#9B7235`。
- 按钮间距不要依赖业务页面手写 margin，应在页头组件内部统一处理。
- 操作区应右对齐，主按钮固定在最右侧。

---

## 5.4 描述列表区域样式

描述列表区域位于标题及操作区域下方。

描述列表与标题及操作区域之间的垂直间距：`16px`  
描述列表排列方式：横向多列 + 纵向换行  

单条描述项高度：`22px`  
单条描述项宽度：`280px`  
描述项标题与描述项内容之间的间距：`0px`  

当前示例中：

- 每行展示 `2` 个描述项。
- 描述项列宽为 `280px`。
- 描述列表整体示例宽度为 `560px`。
- 描述列表整体示例高度为 `52px`。

描述项行间距： `8px`  
描述项列间距： `0px` 

### 描述项标题 / Label 样式

标题文字颜色：`#000000 65%`  
标题文字字号：`14px`  
标题文字字重：`Regular`  
标题文字行高：`22px`  
标题文字字间距：`0px`  

### 描述项内容 / Value 样式

内容文字颜色：`#000000 85%`  
内容文字字号：`14px`  
内容文字字重：`Regular`  
内容文字行高：`22px`  
内容文字字间距：`0px`  

基于 Ant Design 调整时：

- 可基于 `Descriptions` 或自定义描述项布局进行二次封装。
- 若使用 `Descriptions`，需要覆盖默认 label 与 content 的间距、行高、颜色和 item 宽度。
- 描述项不应出现默认表格边框样式。
- 描述项应保持左对齐，不建议居中或右对齐。

---

## 5.5 响应式与溢出规则

组件整体宽度应支持自适应。

标题区域与操作按钮区域并排展示：

- 左侧标题信息优先占据剩余空间。
- 右侧操作按钮保持右对齐。
- 当容器宽度不足时，标题副标题的省略规则：暂不设定
- 当按钮过多或空间不足时，是否收纳为更多操作：收纳

描述列表在宽度不足时的换行规则：标题加内容文字超过18个中文字符，则内容文字的最后一个中文字符用省略号代替

---

## 6. 注意事项

1. 该组件不是普通页面标题，而是详情页对象信息头部，必须同时考虑路径、标题、操作和描述字段的组合关系。
2. 不建议业务页面自行拼接 `Breadcrumb + Title + Button + Descriptions`，应通过组件库统一封装，避免不同页面间距和文字样式不一致。
3. 面包屑、标题、副标题、按钮和描述列表的间距应在组件内部统一控制，不要依赖页面外部样式调整。
4. 主按钮必须位于最右侧，且最多保留一个主操作。
5. 按钮最多展示 4 个，超出时需要业务侧收敛或设计更多操作入口。
6. 描述列表的 Label 使用 `#000000 65%`，Value 使用 `#000000 85%`，不要混用同一颜色。
7. 描述项宽度为 `280px`，需要保持字段排列稳定，避免因字段内容长短造成页面抖动。
8. 当前 SVG 中未明确的 hover、focus、禁用、溢出、省略规则，需要设计师后续补充后再进入组件库实现
