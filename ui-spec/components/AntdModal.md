# Modal 对话框

**中文名称：** 对话框
**英文名称：** Modal
**所属分类：** 反馈
**一句话简介：** 模态对话框，用于承载需要用户交互的重要内容。

---

## 何时使用

当需要用户处理事务、确认操作或填写表单，又不希望跳转页面以致打断工作流程时，使用对话框在当前页面正中打开一个浮层，承载相应的操作。

---

## 基础对话框

基础对话框用于最常见的确认或提示场景，包含标题栏、内容区和底栏按钮区三个模块，适合操作确认、结果反馈、简表单录入等需要用户交互的场景。

该对话框由 Header（标题栏）、Body（内容区）、Footer（按钮区）三个独立模块组成。Header 与 Body 之间通过分隔线区隔，Body 与 Footer 之间也通过分隔线区隔，确保各模块结构清晰、样式分离。

---

## 基础对话框样式

### 整体样式

整体对话框宽度默认 `520px`，居中显示。容器圆角为 `2px`，投影效果为 `0px 12px 48px 16px rgba(0, 0, 0, 0.03)` 叠加 `0px 9px 28px 0px rgba(0, 0, 0, 0.05)` 和 `0px 6px 16px -8px rgba(0, 0, 0, 0.08)`，提供较强的层级浮起感。

---

### Header 模块（标题栏）

标题栏高度为 `56px`，水平内边距为 `24px`，标题使用 `16px`、`Regular` 字重的文字。

标题区域采用白色背景，下方有一条 `1px` 的分隔线，颜色为 `#F0F0F0`，将标题栏与内容区明确区分。右上角提供关闭按钮（叉号图标），点击后可关闭对话框。

关闭按钮颜色为 `#000000`，透明度为 `45%`。

---

### Body 模块（内容区）

内容区高度自适应，水平内边距为 `24px`，垂直内边距为 `30px`。

内容区正文使用 `14px`、`Regular` 字重文字，文字颜色为 `#000000`，透明度为 `65%`，行高为 `1.6`。

内容区在没有特殊需求的情况下（如字段较多），一般采用左右结构布局：左侧为信息标签，右侧为输入框、选择框、单选或多选组件。左侧标签文字使用 `14px`、颜色为 `#000000` 透明度 `65%`，右侧填写内容颜色为 `#000000` 透明度 `85%`。

---

### Footer 模块（按钮区）

按钮区高度为 `52px`，水平内边距为 `24px`，采用右对齐布局。

按钮之间的间距为 `12px`。

按钮区上方有一条 `1px` 的分隔线，颜色为 `#F0F0F0`，将按钮区与内容区分隔。

按钮区一般包含两个按钮：左侧为次要按钮（「取消」），右侧为主要按钮（「确定」），主按钮样式使用品牌金色。

---

## 基础对话框交互规则

对话框通过 `open` 属性控制显示与隐藏。`centered` 保证弹窗每次弹出时水平居中对齐，W+ 页面调用时必须开启。

点击「确定」按钮触发 `onOk` 回调，点击「取消」按钮或右上角关闭按钮触发 `onCancel` 回调。

当提交操作需要等待后端响应时，可使用 `confirmLoading` 为确定按钮提供加载态，避免用户重复点击。

---

## 无底栏弹窗

适用于信息展示、阅读确认等不需要用户做出「确定 / 取消」决策的场景。

该类型对话框无标题栏，无右上角关闭按钮。内容区顶部使用较大的标题文字，字号为 `18px`、字重为 `Bold`，与正文 `14px` 形成明显的层级对比。

底部按钮区域无顶部分隔线，按钮靠左对齐，仅提供一个主要按钮，使用「我知道了」等确认性文案，暗示用户已阅读信息。

内容区垂直内边距为 `20px`，按钮区垂直内边距为 `20px`。

---

## 确认对话框

适用于重要操作前的二次确认，例如删除信息、提交关键数据等不可逆操作。通过警告图标和明确的描述文案，让用户在操作前充分理解操作后果。

该类型对话框无标题栏，无右上角关闭按钮。内容区顶部包含一个黄色警告图标（`ExclamationCircleFilled`，颜色 `#FFA000`，尺寸 `24px`）与标题文字（`18px`、`Bold`）并排展示，图标与标题之间的水平间距为 `12px`。

标题下方为描述文字（`14px`、行高 `1.6`），描述文字与标题之间的垂直间距为 `12px`，描述文字距左侧 `40px` 缩进，与图标对齐。

底部按钮区采用右对齐布局，分隔线无顶部间距。按钮区垂直内边距为 `20px`。

---

## 自定义按钮文案

当业务场景需要特定的确认或取消文案时使用，例如「立即发送 / 稍后处理」「确认提交 / 返回修改」等。

该类型结构同确认对话框，无标题栏、无右上角关闭按钮。按钮区通过自定义 `footer` 渲染业务语义化的按钮文案。

自定义按钮的排列规则保持与基础对话框一致：次要操作在左，主要操作在右，按钮间距为 `12px`。

## 设计规范

### 间距规范

| 模块 | 间距项 | 数值 | 说明 |
|------|--------|------|------|
| 整体 | 容器圆角 | `2px` | 内容区整体圆角 |
| 整体 | 宽度 | `520px`（默认） | 可根据业务需求调整 |
| Header | 标题栏高度 | `56px` | 不含分隔线 |
| Header | 水平内边距 | `24px` | 左右两侧 |
| Header | 标题字号 | `16px` | Regular 字重 |
| Body | 水平内边距 | `24px` | 左右两侧 |
| Body | 垂直内边距 | `30px`（基础） / `20px`（确认类） | 根据弹窗类型调整 |
| Body | 正文字号 | `14px` | Regular 字重，行高 1.6 |
| Footer | 按钮区高度 | `52px` | |
| Footer | 水平内边距 | `24px` | 左右两侧 |
| Footer | 按钮间距 | `12px` | |
| 分隔线 | 线条厚度 | `1px` | 颜色 `#F0F0F0` |

### 投影规范

对话框使用层级投影 `BOX_SHADOW_MODAL`，投影值如下：

```
0px 12px 48px 16px rgba(0, 0, 0, 0.03)
0px 9px 28px 0px rgba(0, 0, 0, 0.05)
0px 6px 16px -8px rgba(0, 0, 0, 0.08)
```

该投影应用于 Modal、Dialog 等需要较强层级感的弹窗场景。

### 字体规范

| 文本类型 | 字号 | 字重 | 颜色 |
|----------|------|------|------|
| 标题栏标题 | `16px` | Regular | 85% #000000 |
| 确认类标题 | `18px` | Bold | 85% #000000 |
| 正文内容 | `14px` | Regular | 65% #000000 |
| 字段标签 | `14px` | Regular | 65% #000000 |
| 字段值 | `14px` | Regular | 85% #000000 |

## 使用场景

**适用场景：**
- 操作确认、结果反馈
- 简表单录入（如新增关联人、完善转介信息）
- 信息展示与阅读确认
- 重要操作的二次确认

**不适用场景：**
- 大量内容展示（请使用 Drawer 或页面跳转）
- 分步骤流程（请使用 `PrivateWorkflowModal`）
- 批量操作面板（请使用 Drawer）

---

## 注意事项

1. `centered` 属性必须开启，保证弹窗居中对齐。
2. 提交操作建议使用 `confirmLoading` 提供加载态，避免用户重复点击。
3. 对话框内容不宜过多，超出内容建议使用抽屉 `Drawer` 或页面跳转承载。
4. 二次确认场景应使用警告图标配合描述文案，让用户充分理解操作后果。
5. 无底栏弹窗仅适合信息确认场景，不适合需要用户决策的操作。
6. 自定义按钮文案应保持简洁，遵循「次要操作在左、主要操作在右」的排列规则。
