# Typography 字体规范

**中文名称：** 字体规范
**英文名称：** Typography
**所属分类：** 全局样式
**一句话简介：** 统一私行组件库中的字体栈、字号、行高、数字字体、图标尺寸和文字颜色规则，以 14px 正文和 CMBSans 数字体系为基准。

---

# Typography 文字

## 何时使用

文字规范用于统一产品界面中的字体选择、字号层级、行高关系、字重使用、文字颜色和排版规则。适用于移动端、PC 端、管理后台、客户展示页、营销活动页等界面中的标题、正文、辅助说明、数字信息、标签、按钮和提示文案。

文字系统应保证信息层级清晰、阅读节奏稳定、重点内容突出，并在不同设备和业务场景中保持一致的视觉表现。文字规范不仅用于控制视觉样式，也用于帮助用户快速识别页面结构、理解业务信息和完成操作。

---

## 字体 / Font Family

字体体系用于保证中文、西文、数字和特殊符号在不同系统环境下具有一致的显示效果。字体选择应优先使用系统默认字体，确保加载稳定、兼容性强，并降低页面性能成本。

### 中文字体

中文内容优先使用系统无衬线字体。中文字体应具备良好的屏幕可读性，适合金融业务中较长的说明文字、表单字段、列表信息和数据说明。

在不同终端中，中文字体可按照系统环境自动调用：

- iOS / macOS 环境优先使用 `PingFang SC`。
- Windows 环境优先使用 `Microsoft YaHei`。
- Android 环境优先使用系统默认中文字体。

### 西文字体

西文内容优先使用系统无衬线字体。西文字体应与中文字体保持相近的字面高度和视觉重心，避免中英文混排时出现跳动感。

在英文标题、按钮、状态标签、字段名称、代码类信息等场景中，应保持字母间距自然，不建议随意扩大字间距。

### 数字字体

数字内容应用于金额、比例、日期、数量、统计指标等数据场景。数字字体应保证字符宽度稳定、识别清晰，避免不同数字宽度差异过大影响列表和图表对齐。

在金融场景中，关键金额和统计数字可适当提高字号或字重，但不应过度放大，以免破坏页面整体平衡。

### 字体定义

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Helvetica Neue", Helvetica, Arial, sans-serif;
```

> **兜底方案**：若系统所有字体均不可用，必须使用 `Microsoft YaHei` 字体作为显示字体。


---

## 字号与行高 / Font Sizes & Line Height

字号与行高用于建立页面信息层级。字号越大，信息权重越高；行高应根据字号同步调整，保证文字在多行展示时具有足够的阅读空间。

文字字号建议采用偶数递增规则，常用字号集中在 `12px`、`14px`、`15px`、`16px`、`18px`、`20px`、`24px`、`28px`、`32px`。在同一页面中不宜使用过多字号，避免层级混乱。

---

### 页面标题

页面标题用于表达当前页面或模块的核心主题，通常出现在页面顶部、卡片标题区或重要内容区。

页面标题应具备较强识别度，字号建议为 `20px` 或以上，`Bold`，字重500。标题文字不宜过长，若标题包含业务对象或状态说明，应优先保证主标题简洁明确。

---

### 主标题

主标题用于页面中的一级信息层级，如业务模块名称、关键分组标题、重要结果标题等。

主标题建议使用 `18px` / `26px`，`Bold`，字重500。当主标题承载金融业务关键信息时，可配合较高对比度文字颜色使用。

---

### 次级标题

次级标题用于模块内分组、卡片标题、表单分区标题等场景。

次级标题建议使用 `16px` / `24px`，`Bold`，字重500。次级标题应比正文更醒目，但不应抢占页面主标题层级。

---

### 正文文字

正文文字用于承载主要阅读内容，包括说明文字、段落内容、表单说明、列表信息等。

正文文字建议使用 `14px` / `22px`，字重为 `Regular`。正文应保持稳定的行高和适中的行长，避免过密排版造成阅读压力。

---

### 辅助文字

辅助文字用于展示补充说明、提示信息、字段解释、弱化说明等内容。

辅助文字建议使用 `12px` / `20px` 或 `14px` / `22px`。辅助文字颜色应低于正文文字权重，避免与主要信息竞争。

---

### 数字文字

数字文字用于金额、收益率、数量、评分、资产规模等需要突出识别的数据内容。

关键数字可使用 `24px`、`28px`、或`32px` ，行高可对应为 `32px`、`36px`、或`40px` 或。数字信息应根据业务重要程度设置字号，不建议所有数字统一放大。

---

### 标签文字

标签文字用于状态标签、分类标签、业务属性、风险等级、审批状态等场景。

标签文字建议使用 `12px` / `20px` 。标签文字应简短明确，避免出现长句；若标签文字过长，可采用省略或换行策略。

---

### 按钮文字

按钮文字用于表达用户可执行操作，应简洁明确，避免使用模糊词。

按钮文字建议使用 `14px` / `22px`，重要按钮可使用 `Regular` 字重。按钮文字应与按钮尺寸匹配，保证垂直居中和视觉稳定。

---

## 图标大小 / Icon Size

图标大小用于统一界面中功能图标、状态图标、操作图标和提示图标的视觉比例。图标应与文字字号匹配，避免出现图标过大或过小的问题。

### 小图标

小图标用于表格操作、输入框后缀图标、辅助说明图标、状态辅助图标等场景。

小图标建议尺寸为 `12px` 或 `16px`，适合与 `12px`、`14px` 文字配合使用。

### 常规图标

常规图标用于按钮图标、列表图标、表单项图标、导航图标等通用场景。

常规图标建议尺寸为 `16px`，适合与正文和按钮文字配合使用。

### 强调图标

强调图标用于重要状态、模块入口、信息提示、重点功能等场景。

强调图标建议尺寸为 `24px`。当图标与标题组合使用时，应保证图标视觉高度与文字行高协调。

### 大图标

大图标用于空状态、结果页、引导页、业务卡片入口等场景。

大图标可使用 `24px`。大图标应搭配简洁说明文字使用，不宜与大量正文同时出现。

---

## 字体颜色 / Font Colors

字体颜色用于表达信息层级、状态语义和可交互属性。文字颜色应遵循主次分明、对比充足、状态清晰的原则。

### 主要文字

主要文字用于页面标题、正文重点内容、表单字段值、列表主信息等高优先级内容。

主要文字颜色建议使用黑色透明度 `85%`，即 `rgba(0, 0, 0, 0.85)`。在深色背景中，主要文字建议使用白色透明度 `100%`。

### 次要文字

次要文字用于辅助信息、说明内容、次级字段、补充文案等场景。

次要文字颜色建议使用黑色透明度 `65%`，即 `rgba(0, 0, 0, 0.65)`。在深色背景中，次要文字建议使用白色透明度 `65%`。

### 占位文字

占位文字用于输入框 placeholder、待输入提示、未填写状态等场景。

占位文字颜色建议使用黑色透明度 `45%`，即 `rgba(0, 0, 0, 0.45)`。占位文字不应与已输入内容使用相同颜色，避免用户误判。

### 禁用文字

禁用文字用于不可点击、不可编辑、不可选择的文字内容。

禁用文字颜色建议使用黑色透明度 `25%`，即 `rgba(0, 0, 0, 0.25)`。禁用态文字应明显弱化，但仍需保持基本可识别。

### 链接文字

链接文字用于可点击跳转、查看详情、文本按钮等场景。

链接文字建议使用品牌蓝色 `#1874FF`。链接文字应具备明确可点击感，可在 hover 或 active 状态下通过颜色变化表达反馈。

### 成功文字

成功文字用于表达完成、正常、通过、成功等正向反馈。

成功文字建议使用绿色 `#00C078`，常用于状态标签、提示信息、结果反馈等场景。

### 警告文字

警告文字用于表达待确认、需关注、存在风险但不阻断操作的状态。

警告文字建议使用橙色 `#FFA000`。警告文字不宜大面积使用，避免造成页面紧张感。

### 错误文字

错误文字用于表达失败、异常、校验不通过、风险阻断等需要立即处理的信息。

错误文字建议使用红色 `#FF4D4F`。错误文字应配合清晰的问题说明和处理建议使用。

---

## 状态文字颜色

状态文字用于统一不同业务状态下的视觉表达。状态色应与图标、标签、提示组件保持一致，避免同一状态在不同页面中使用不同颜色。

| 类型 | 颜色 | 透明度 | 使用说明 |
|------|------|----------|
| Success 成功 | `#00C078` | 100% | 操作成功、审批通过、状态正常 |
| Info 信息 | `#000000` | 65% | 常规说明、系统信息、引导提示 |
| Warning 警告 | `#FFA000` | 100% | 风险提示、待确认、需关注 |
| Error 错误 | `#FF4D4F` | 100% | 操作失败、校验错误、风险阻断 |
| Link 链接 | `#1874FF` | 100% | 文本链接、查看详情、跳转入口 |

## 状态文字优先级

页面中文字应根据业务重要程度进行层级区分。优先级越高的文字应使用更高的字号、字重或颜色对比度；优先级越低的文字应降低颜色透明度或缩小字号。

### 高优先级文字

用于页面标题、核心指标、重要金额、风险结论、操作结果等内容。高优先级文字应确保用户第一眼可识别。

### 中优先级文字

用于正文内容、列表主信息、表单字段值、模块说明等内容。中优先级文字应保持稳定可读，不应与高优先级文字竞争。

### 低优先级文字

用于辅助说明、备注、占位提示、次要标签等内容。低优先级文字应适当弱化，避免干扰用户完成主要任务。

## 排版与可读性规则

### 行高规则

行高应大于字号，保证文字在多行展示时具有足够呼吸感。正文行高建议为字号的 `1.5` 倍左右，标题行高可略紧凑。

### 字重规则

字重应服务于信息层级，不应大面积使用加粗。标题、关键数字、重要状态可使用 `Bold，字重500` ；正文和辅助说明应使用 `Regular`。

### 对齐规则

列表、表格、表单等结构化信息应保持左对齐。金额、数量等数据在表格中可根据业务需要右对齐，以便用户快速比较。

### 换行规则

标题和按钮文案应尽量避免换行。正文说明可自然换行，但应控制单段长度。标签和状态文案过长时，应优先使用省略或缩短表达。

### 中英文混排

中英文、数字和符号混排时，应保持统一字体体系和自然间距。金额、百分比、单位等内容应作为整体展示，避免数字与单位断行。

## 使用场景

**适用场景：**

- 页面标题、模块标题、卡片标题等标题体系
- 正文说明、业务规则、操作说明等阅读内容
- 表单字段、列表信息、表格内容等结构化信息
- 金额、比例、数量、资产规模等数字信息
- 状态标签、提示文案、链接文字和按钮文字

**不适用场景：**

- 品牌海报、营销主视觉等强视觉创意场景
- 插画、图标、Logo 等非文字规范内容
- 需要特殊字体授权的品牌字体场景

---

## 注意事项

1. 同一页面中字号层级不宜过多，建议控制在 3 个主要层级内。
2. 正文文字应优先保证可读性，不建议使用过浅颜色或过小字号。
3. 金额和关键数字可以放大，但应根据业务重要程度分层使用。
4. 辅助文字不应承载关键决策信息，关键提示应提升到正文或警告层级。
5. 状态色应保持语义一致，成功、信息、警告、错误不应混用。
6. 深色背景中的文字应使用白色透明度体系，避免直接沿用浅色背景中的黑色透明度。
7. 中英文和数字混排时，应优先保证对齐、识别和断行稳定。
8. 禁用文字应表达不可操作状态，不应仅依赖颜色，也可配合禁用按钮、禁用图标等视觉状态。
