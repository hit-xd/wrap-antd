# Checkbox

**中文名称：** 多选框
**英文名称：** Checkbox
**所属分类：** 数据录入
**一句话简介：** 在一组可选项中进行多项选择，支持半选状态。

---

## 何时使用

Checkbox 用于在一组选项中进行多项选择，用户可以同时选择一个或多个结果。适用于表单填写、条件筛选、配置项、批量操作、偏好设置等场景。

Checkbox 强调「从多个候选项中选择若干项」。当所有选项都需要直接展示，且选项数量不多时，Checkbox 比 Select 更直观；当选项过多、页面空间有限、需要搜索或分组时，应使用 Select、TreeSelect、Transfer 或筛选面板。

适用于以下情况：

- 在一组选项中选择多个结果。
- 选项数量较少，通常为 `2-8` 个。
- 所有选项需要直接展示，便于用户快速比较。
- 选项之间不是互斥关系，可以同时成立。
- 表单项、筛选项、配置项中需要选择多个条件。
- 需要实现「全选 / 部分选中 / 取消全选」等联动场景。
- 选项含义能够通过简短文字、说明 icon 或辅助说明表达清楚。

不建议在以下情况中使用：

- 只能选择一个结果时，应使用 Radio。
- 需要即时开启 / 关闭某个功能时，应使用 Switch。
- 选项数量很多、需要搜索或收起展示时，应使用 Select、TreeSelect 或 Transfer。
- 需要从日期、时间、金额范围中选择时，应使用对应的专业组件。
- 选项含义复杂，需要大量解释或多步骤判断时，应拆分为独立配置区域。
- 多个条件之间存在复杂依赖关系时，不应全部堆叠为 Checkbox，应使用筛选面板或配置向导。

## 基础多选框

基础多选框用于最常见的多项选择场景。它由方形选择框和选项文字组成，适用于表单、筛选条件、配置项、弹窗选项等轻量多选场景。

基础多选框应保持轻量，不应承载过多装饰。选项文案应短、明确、可比较，避免使用含糊表达。

示例：

- 短信通知 / 邮件通知 / 站内提醒
- 电话 / 面访 / 短信 / 活动
- 高净值客户 / 私行客户 / 潜力客户
- 风险提示 / 到期提醒 / 自动同步
- 可读 / 可编辑 / 可导出

---

## 基础多选框结构

基础多选框由以下元素组成：

- **选择框**：方形多选控件，用于展示未选中、选中、半选、禁用等状态。
- **选项文字**：位于选择框右侧，用于说明该选项的含义。
- **后置说明 icon**：可选元素，位于选项文字右侧，用于承载业务解释、规则说明或限制说明。
- **Tooltip 气泡说明**：可选元素，当用户 Hover 后置说明 icon 时展示。

基础 Checkbox 中，选择框和选项文字应垂直居中对齐。用户点击选择框或选项文字均应触发选中 / 取消选中。

## 常规样式

### 组件尺寸

常规 Checkbox 选项内容高度为 `24px`。为保证可点击性，实际点击热区高度不应小于 `32px`。

样式规格：

| 项目 | 数值 | 说明 |
|---|---:|---|
| 选项内容高度 | `24px` | 选择框与文字的视觉内容高度 |
| 建议点击热区高度 | `≥32px` | 点击区域应覆盖选择框和文字 |
| 选择框尺寸 | `16px × 16px` | 方形 Checkbox 外框尺寸 |
| 选择框圆角 | `2px` | 保持轻量方形特征 |
| 勾选线宽 | `2px` | 选中态白色勾选标记 |
| 半选横线尺寸 | `8px × 2px` | 半选态内部横线 |
| 选择框与文字间距 | `8px` | 选择框右侧到文字左侧的距离 |
| 选项文字字号 | `14px` | 与表单正文一致 |
| 选项文字行高 | `24px` | 保证与选择框垂直居中 |
| 字间距 | `0px` | 不额外增加字距 |
| 横向选项间距 | `24px` | 同组横向排列时选项之间的距离 |
| 纵向选项间距 | `8px` | 同组纵向排列时选项之间的距离 |

### 未选中状态

未选中状态用于表示当前选项未被选择。

样式规则：

- 选择框背景色：`#FFFFFF`
- 选择框边框色：`#000000`，透明度 `25%`
- 选项文字颜色：`#000000`，透明度 `85%`
- 后置说明 icon 颜色：`#000000`，透明度 `45%`

### Hover 状态

Hover 状态用于提示用户当前选项可操作。

样式规则：

- 鼠标指针：`pointer`
- 选择框边框色：`#9B7235`
- 选择框背景色：`#FFFFFF`
- 选项文字颜色：`#000000`，透明度 `85%`
- 后置说明 icon 可变为 `#9B7235`

Hover 应作用于整个选项点击区域，而不仅是方形选择框。

### 选中状态

选中状态用于表示当前选项已被选择。

样式规则：

- 选择框背景色：`#9B7235`
- 选择框边框色：`#9B7235`
- 勾选标记颜色：`#FFFFFF`
- 选项文字颜色：`#000000`，透明度 `85%`

常规 Checkbox 选中后不建议改变选项文字颜色，避免整组文本权重跳动。状态识别主要依赖选择框状态。

### 半选状态

半选状态用于表示当前节点下存在部分选中内容，常见于「全选」或层级选择场景。半选状态只表达选择状态，不代表当前项自身一定已完整选中。

样式规则：

- 选择框背景色：`#9B7235`
- 选择框边框色：`#9B7235`
- 半选横线颜色：`#FFFFFF`
- 半选横线尺寸：`8px × 2px`
- 选项文字颜色：`#000000`，透明度 `85%`

使用规则：

- 全选项下有部分子项被选中时，全选项展示半选状态。
- 用户点击半选项时，应根据业务逻辑切换为全选或取消全选。
- 半选状态通常由父子联动计算得出，不建议作为普通业务选项的独立状态。

### 禁用未选中状态

禁用未选中状态用于表示当前选项不可选择，且未被选中。

样式规则：

- 选择框背景色：`#F4F6F8`
- 选择框边框色：`#000000`，透明度 `25%`
- 选项文字颜色：`#000000`，透明度 `25%`
- 后置说明 icon 颜色：`#000000`，透明度 `25%`
- 鼠标指针：`not-allowed`
- 不展示 Hover 反馈

### 禁用选中状态

禁用选中状态用于表示当前选项已被选中，但用户不可修改。

样式规则：

- 选择框背景色：`#F4F6F8`
- 选择框边框色：`#000000`，透明度 `25%`
- 勾选标记颜色：`#000000`，透明度 `25%`
- 选项文字颜色：`#000000`，透明度 `25%`
- 鼠标指针：`not-allowed`
- 不展示 Hover 反馈

## 后置说明 icon

当多选项需要解释业务规则、专业术语、适用范围或限制条件时，可在选项文字后增加说明 icon。

适用场景：

- 选项含义存在专业术语，用户可能不理解。
- 选项会影响后续流程、权限、数据范围或业务规则。
- 选项之间差异较细，需要补充说明。
- 说明内容较短，适合通过 Tooltip 气泡展示。

不适用场景：

- 每个选项都需要长篇解释。
- 说明内容包含复杂表格、流程或多段文本。
- 说明内容是必读信息，不应隐藏在 Tooltip 中。

样式规则：

| 项目 | 数值 | 说明 |
|---|---:|---|
| 说明 icon 尺寸 | `16px × 16px` | 位于选项文字右侧 |
| 文字与说明 icon 间距 | `8px` | 保持说明 icon 与文字关联 |
| 默认 icon 颜色 | 45% `#000000` | 弱提示 |
| Hover icon 颜色 | `#9B7235` | 悬停时强化可查看状态 |
| Tooltip 最大宽度 | `280px` | 避免气泡过宽 |

Tooltip 应在用户 Hover 到说明 icon 时展示，而不是 Hover 整个 Checkbox 选项时展示。

## Checkbox.Group 与全选联动

Checkbox.Group 用于组织一组多选项，并统一输出选中值列表。W+ 封装层应保留 Ant Design 的 `options`、`value`、`defaultValue`、`disabled`、`onChange` 等能力。

当业务中存在「全选」或父子联动时，应使用半选状态表达部分选中关系。

全选联动规则：

- 子项全部未选中：全选项为未选中状态。
- 子项部分选中：全选项为半选状态。
- 子项全部选中：全选项为选中状态。
- 点击未选中或半选的全选项：默认选择全部子项。
- 点击已选中的全选项：默认取消全部子项。

不建议在没有父子关系或全选语义的普通多选项中使用半选状态。

## 卡片式多选框

卡片式多选框用于需要更强选项边界、需要承载图标、辅助说明或需要突出当前选择结果的场景。它本质上仍是多选，只是将选项视觉封装为卡片或按钮形态。

卡片式 Checkbox 可基于 Ant Design Checkbox 的自定义 option 渲染实现。W+ 封装层应统一卡片高度、边框、文字、图标、说明和状态样式。

适用于以下情况：

- 选项需要更强视觉边界。
- 选项之间需要横向对比。
- 选项需要搭配业务 icon。
- 选项需要展示辅助说明文字。
- 多选结果会影响后续表单内容或策略配置。
- 页面希望减少方形 Checkbox 的视觉噪音。

不建议在以下情况中使用：

- 选项非常多，超过 `8` 个且需要大量换行展示。
- 选项文案非常长，卡片宽度难以控制。
- 选项之间不需要强化边界，普通 Checkbox 已足够清晰。
- 只能单选时，不应使用卡片式 Checkbox，应使用 Radio Card。

---

## 卡片式多选框结构

卡片式 Checkbox 可由以下元素组成：

- **卡片容器**：承载选项内容，并通过边框、背景色和文字颜色表达状态。
- **前置图标**：可选元素，用于表达选项类型，例如电话、面访、开户、活动等。
- **选项文字**：选项主文案。
- **后置说明 icon**：可选元素，用于展示业务解释。
- **辅助说明文字**：可选元素，位于选项主文案下方，用于解释适用范围或限制条件。
- **选中角标**：可选元素，位于卡片右下角，用于强化已选状态。

卡片式 Checkbox 不建议再展示方形 Checkbox 选择框，避免视觉重复。选中态通过卡片边框、文字颜色、浅色背景或角标表达。卡片式 Checkbox 不展示半选状态。

## 卡片式多选框样式

### 基础卡片尺寸

| 项目 | 数值 | 说明 |
|---|---:|---|
| 基础卡片高度 | `32px` | 单行卡片选项推荐高度 |
| 卡片圆角 | `2px` | 与 Input、Select 等表单组件保持一致 |
| 卡片边框 | `1px` | 通过边框表达状态 |
| 左右内边距 | `16px` | 文字与卡片边界距离 |
| 上下内边距 | `4px` | 保证 `32px` 高度内垂直居中 |
| 文字字号 | `14px` | 与表单正文一致 |
| 文字行高 | `24px` | 保持单行可读性 |
| 前置图标尺寸 | `16px × 16px` | 业务图标尺寸 |
| 图标与文字间距 | `8px` | 前置图标到选项文字的距离 |
| 文字与后置说明 icon 间距 | `8px` | 文字末尾到说明 icon 视觉边界的距离，不含 Tooltip 热区 |
| 多个卡片横向间距 | `8px` | 同组卡片之间的水平间距 |
| 多个卡片纵向间距 | `8px` | 换行或纵向排列时的垂直间距 |

### 卡片未选中状态

- 卡片背景色：`#FFFFFF`
- 卡片边框色：`#000000`，透明度 `25%`
- 选项文字颜色：`#000000`，透明度 `85%`
- 前置图标颜色：`#000000`，透明度 `45%`
- 后置说明 icon 颜色：`#000000`，透明度 `45%`

### 卡片 Hover 状态

- 鼠标指针：`pointer`
- 卡片边框色：`#9B7235`
- 卡片背景色：`#FFFFFF`
- 选项文字颜色：`#000000`，透明度 `85%`
- 前置图标和后置说明 icon 可变为 `#9B7235`

### 卡片选中状态

- 卡片背景色：`#FFFFFF`
- 卡片边框色：`#9B7235`
- 选项文字颜色：`#9B7235`
- 前置图标颜色：`#9B7235`
- 右下角可展示选中角标，角标颜色为 `#9B7235`

卡片选中态应通过边框与文字颜色共同表达。若业务页面中选项密度较高，可只保留边框与角标，避免文字颜色过多跳动。

### 卡片不展示半选状态

卡片式 Checkbox 仅展示未选中、选中、禁用未选中、禁用选中四类状态，不展示半选状态。

半选状态只用于常规 Checkbox 的全选、父子联动或层级选择场景。若卡片式多选存在部分选中含义，应通过上层分组的常规 Checkbox、辅助说明文字或业务状态文案表达，不应在单个卡片上展示半选角标或半选横线。

### 卡片禁用状态

- 卡片背景色：`#F4F6F8`
- 卡片边框色：`#000000`，透明度 `25%`
- 选项文字颜色：`#000000`，透明度 `25%`
- 图标颜色：`#000000`，透明度 `25%`
- 鼠标指针：`not-allowed`
- 不展示 Hover 反馈

### 带辅助说明的卡片尺寸

当卡片中包含辅助说明文字时，卡片高度应随内容增加，不应强行压缩到 `32px`。

建议规则：

| 场景 | 建议高度 | 说明 |
|---|---:|---|
| 单行主文案 | `32px` | 只展示选项文字 |
| 主文案 + 一行说明 | `64px` | 适合短说明 |
| 主文案 + 两行说明 | `80px` 起 | 说明内容较长时使用 |

辅助说明文字样式：

- 字号：`12px`
- 行高：`20px`
- 颜色：`#000000`，透明度 `45%`
- 与主文案垂直间距：`2px`
- 说明文字过长时可换行，不建议横向溢出

## 前置图标

卡片式 Checkbox 可使用前置图标辅助表达选项类型。常见于触达方式、业务类型、客户标签、活动类型等场景。

样式规则：

- 前置图标尺寸：`16px × 16px`
- 图标圆角：按图标自身风格控制，底色图标圆角建议 `4px`
- 图标与文字间距：`8px`
- 默认图标颜色：`#000000`，透明度 `45%`
- 选中态图标颜色：`#9B7235`
- 禁用态图标颜色：`#000000`，透明度 `25%`

如果图标本身带品牌色或业务色，应降低饱和度，避免与选中主色冲突。图标只用于辅助识别，不应替代选项文字。

## 后置说明 icon 与辅助说明文字

### 后置说明 icon

当卡片选项较短，但含义存在专业解释、业务规则、适用范围或限制条件时，可在选项文字右侧增加说明 icon。

样式规则与常规 Checkbox 一致：

- 说明 icon 尺寸：`16px × 16px`
- 选项文字与后置说明 icon 的水平间距：`8px`
- 该 `8px` 为最终视觉间距，不应叠加通用说明 icon 的左外边距；实现时建议在卡片内部使用父级 `gap: 8px`，并将说明 icon 的 `margin-left` 置为 `0`
- 该间距指文字末尾到 icon 视觉边界的距离，不包含 Tooltip 触发热区
- 默认颜色：45% `#000000`
- Hover 颜色：`#9B7235`
- Tooltip 最大宽度：`280px`

### 辅助说明文字

当说明内容需要直接被用户看到时，应使用辅助说明文字，而不是隐藏在 Tooltip 中。

适用场景：

- 用户需要直接比较不同选项。
- 说明内容会影响选择判断。
- 选项之间差异较大，需要简短解释。
- 说明文字不超过两行。

不适用场景：

- 说明内容很长，需要完整段落解释。
- 说明内容包含复杂规则、表格或公式。
- 说明内容不是当前选择的必要信息。

---

## 排列规则

### 横向排列

适用于选项数量少、文案短、需要快速比较的场景。

规则：

- 常规 Checkbox 横向间距为 `24px`。
- 卡片 Checkbox 横向间距为 `8px`。
- 横向排列时应保持同一行垂直居中。
- 当选项过多导致换行时，每行之间垂直间距为 `8px`。

### 纵向排列

适用于选项文案较长、需要阅读说明、选项数量较多或页面窄的场景。

规则：

- 常规 Checkbox 纵向间距为 `8px`。
- 卡片 Checkbox 纵向间距为 `8px`。
- 纵向排列时应保持左侧对齐。
- 带辅助说明的卡片建议纵向排列，减少横向挤压。

---

## 交互规则

### 选择与取消

用户点击 Checkbox 后，该选项在选中和未选中之间切换。Checkbox.Group 应同步更新选中值列表。

### 多选组

同组 Checkbox 可同时选择多个结果。选项之间不应存在单选互斥关系。如果存在互斥关系，应改用 Radio。

### 全选与半选

存在全选项时，应根据子项选中情况展示未选中、半选或选中状态。半选状态应由业务数据计算，不建议用户直接手动配置。

### 禁用

禁用状态下不可点击、不可聚焦，也不应出现 Hover 或选中反馈。禁用选项仍可用于展示当前已存在的不可修改结果。

### Tooltip

说明 icon 的 Tooltip 应在 Hover 到 icon 热区时展示，不应覆盖整个选项 Hover。Tooltip 文案应简短，优先解释影响、限制或业务规则。

## 使用场景

**适用场景：**

- 表单中的多项选择。
- 列表、表格、客户池等多条件筛选。
- 批量操作条件勾选。
- 偏好设置、通知方式、触达渠道选择。
- 权限范围、数据范围、客户标签选择。
- 有全选、部分选中、父子联动的选择场景。
- 需要直接展示少量候选项供用户比较的场景。

**不适用场景：**

- 只能选择一个结果，应使用 Radio。
- 即时开启 / 关闭功能，应使用 Switch。
- 选项数量过多，应使用 Select、TreeSelect、Transfer 或筛选面板。
- 需要输入自定义内容，应结合 Input 或使用独立表单项。
- 选项解释过长，不应全部塞入 Checkbox，应拆成配置区域。
- 复杂权限矩阵不应只使用普通 Checkbox，应使用表格或权限树。

## 注意事项

1. Checkbox 用于多项选择，不应承载互斥单选逻辑。
2. Checkbox 通常用于状态标记或多项选择，往往需要配合「确认 / 提交 / 保存」后生效。
3. 单独使用 Checkbox 表达二元状态时，应确认是否需要即时生效；若需要即时生效，应优先使用 Switch。
4. 选项文案应简短、明确，不建议使用「选项一」「配置项」等模糊文案。
5. 常规 Checkbox 适合轻量选择；卡片式 Checkbox 适合需要强化边界、展示 icon 或说明的场景。
6. 半选状态应主要用于全选或父子联动，不建议作为普通选项状态。
7. 后置说明 icon 只用于短说明，不应承载必读或复杂说明。
8. 辅助说明文字应直接帮助用户判断，不应重复主文案。
9. 禁用状态下不可点击，也不应出现 Hover 或选中反馈。
10. 同一组 Checkbox 的视觉形态应保持一致，不建议常规样式和卡片样式混用。
11. Checkbox、Radio、Switch 的边界应清晰：多选标记用 Checkbox，单项选择用 Radio，即时开关用 Switch。
